What you will experience

High-level request/data flow for this demo: A single Nginx container acts as the reverse proxy, proxying static asset requests to a separate static-content container and routing /api/* to the backend processing server.

  • Nginx (Docker) acts as the single entry point — proxies static asset requests to the Static Content container and routes API calls to the Processing Service container.
  • Static Content (Docker) serves the built HTML/JS/CSS/model files for the UI.
  • Processing Service (Docker) ingests data, executes the CV pipeline, and deliver results via email.
  • Browser loads the UI, performs on-device video processing with TFJS, handles user interaction, and calls the API for backend tasks.
